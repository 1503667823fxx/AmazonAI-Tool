## 文档维护

### 更新原则
1. **及时更新** - 功能变更时同步更新相关文档
2. **版本控制** - 重要变更保留历史版本到 archive 目录
3. **用户友好** - 面向用户的文档使用简洁明了的语言
4. **技术准确** - 技术文档保持准确性和完整性

### 文档分类
- **用户指南** - 面向最终用户的使用说明
- **技术文档** - 面向开发者的技术文档
- **归档文档** - 历史文档和开发过程记录

### 贡献指南
1. 新增文档请放在对应分类目录下
2. 更新现有文档时保留重要历史版本
3. 使用清晰的文档结构和格式
4. 及时更新本 README 文档的索引
